说实话，在写这篇文章前的月余前，我还是个被 AI 时代冲击的焦虑码农，印象是从所谓全民「养龙虾」开始。那一阵子好像对什么都提不起兴趣，体重也从 140+ 瘦到 120 多，和大学的时候一模一样，是因为 30 岁了吗。2026 的上半年，好像来了一波身心的双重低谷。

慢慢感觉到不对，才开始改变。这里来说说关于 AI 的吧。

上文提到「养虾」，到现在我也没养过，不过还是在豆包那里深入了解一番，什么日常任务自动化、管理本地文件夹，于我而言都是看似有用其实没啥用的。至于 Vibe Coding，这个其实一直以来就有。第一次深度使用是在去年做了个今日热榜的小玩具，到后来使用 Astro 完全重构个人网站，也算是半个 Vibe Coding 者吧。工作上一直使用公司的企业通义灵码，主要是作为 IDEA 和 VSCode 里的插件使用。但是从 Claude Code 爆火开始，什么 CLI，什么从命令行使用，我就彻底落伍了。再后来 Codex，这类开发智能体功能愈来愈强大，甚至到了桌面版，好像落下了太多了。

想起之前从一个 Java 程序员转半吊子全栈的时候，什么尚硅谷、黑马的学习视频都是从 B 站开始的，这次也可以啊。于是就在 B 站看了一堆 Claude Code、Codex 的视频，开始追赶先行一步的同行者。

各家大模型厂商百花齐放，还有就是在国内原生使用 CC、Codex 是需要 FQ 的，于是诞生了一类 LLM 统一 API 聚合网关。尚硅谷这类视频里基本都推荐 OpenRouter，在朋友博客见到的 ZenMux，此外还有其他 AI 工具比如  CC Switch 等，认识了很多新玩意儿。

现在我的 AI 使用方案很简单：

首先是 **豆包** ，负责日常技术点问答，快速了解一个东西，还有就是基本完全替代了之前查技术博客的方式。当然不止技术方面，其他方面也会在此先了解个大概。此外，某个想法萌芽，也会先在豆包讨论可行性、技术方案等，初步确定大致方向。想深入或者实现的话就交给下面要提到的 CC + DeepSeek。

其次是 Claude Code + DeepSeek，这个主要是个人使用。做好一些全局配置后，直接在 VSCode、IDEA 或 WebStorm 这类软件里以插件方式使用。第一件事就是使用 Nextjs 将个人网站重构，Astro 也很好，不过随着工作和一些其它原因需要将 Nextjs 纳入到技术栈，最终效果还不错，其实还是朋友的 [网站](https://varzy.me/) 做得好，参考性强 😆。

关于桌面版，当初 Cursor、Windsurf 大行其道的时候，尝试过字节的 Trea，最初 Astro 版本的个人网站就基于 Trea 实现。Codex 桌面版改版后爆火的时候，我也懒得去折腾了。Claude Code 桌面版也根本没去考虑。小试了一下智谱的 ZCode，感觉一般，可能是因为早期版本，后面功能迭代就不知道了。后来工作上使用公司订阅的国际版 Qoder，最初也是着实一般，不过随着功能迭代，越来越好用了，尤其 Repo Wiki 功能，虽生成的时候耗费 Credits 巨大，好在是一次性的。个人使用不敢使用公司订阅的 Qoder，后续观望下，OpenCode、腾讯的 WorkBuddy 或者订阅 Qoder 个人版…

我觉得对于开发人员来说，有两种人是有天然优势的。一个是了解多个技术栈，不一定很熟悉或精通，并且对技术趋势或前沿咨询有兴趣保持不断认识的人。还有就是具有产品思维，除了完成工作编码本身，还时不时有一些新的想法，冒一些有趣点子的人。如虎添翼，对于这两类人 AI 自然就成了使自身更加强大的双翼。

随着这阵子，工作上需要对 Datus Agent 深入了解，有了个契机去了解 Agent 智能体开发，学习是没有头的。就像 Java 语言设计和 SpringBoot 这种基础性框架一样，大模型的设计和 LangChain、LangGragh 这种基础性框架的开发是需要高端人才去搞的，但是在此之上的应用端，不管是工作上还是个人方面，都希望能赶上车，留点痕迹下来。

AI 可以带来焦虑，但当你张开怀抱迎接它，将是如虎添翼。2026.7.19 晨 于津家中记
